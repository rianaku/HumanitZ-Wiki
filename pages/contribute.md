## <span style="color:#E79543;">1. Contribution Workflow</span>

<div style="background:#111; padding:16px; border:1px solid #333; border-radius:8px;">

1. **Fork** this repo to your own GitHub account.  
2. **Create a branch** for your changes (e.g. `feature/add-weapons-data`).  
3. **Edit or add .md files** in the root (or `/docs` if using GitHub Pages).  
4. **Commit** with a clear message.  
5. **Open a Pull Request** back to this repo with a short summary of what you changed.  

</div>

---

## <span style="color:#E79543;">2. Style Guidelines</span>

- Use **Markdown** + light HTML only (no scripts).
- Keep headings consistent with existing pages.
- Use the **navigation bar** at the top of new pages.
- Use **HumanitZ orange** (`#E79543`) for major headings via `<span>` or inline style.
- Use tables for items, stats, and recipes where possible.
- Use `<details>` for spoiler-heavy or very long sections.

---

## <span style="color:#E79543;">3. Content Guidelines</span>

- Prefer **tested, confirmed information**.
- If something is speculative, mark it clearly as such.
- For patch-specific behaviour, note the **game version**.
